We use a CNN-LSTM architecture for this module, wherein the convolution layer extracts local features, while LSTM focuses on learning the long-term sequential dependency from the encoded characters. For pooling between the convolution layers, we use MaxPooling. We place a dense layer with multiple dense units after LSTM to aggregate the outputs from LSTM units (e.g., from 512 LSTM units to 256 dense units), and use another dense layer with a single unit to make a single output value representing the prediction probability. The network returns predicted probability for each individual URL. For obtaining a prediction value for an email with multiple URLs, we take the highest prediction probability of all embedded URLs in the email. The rationale for this is, presence of one phishing URL is sufficient to classify a mail as a phishing attempt.
